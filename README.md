# 九宫格决策

#### 思路1：
* 文件：strategy_test 是九宫格决策思路1的代码
* 编辑行为与状态：
    数据编辑逻辑:32位数据，以每两位为一个单元，分为16个单元，标记为0~15
  * 行为编辑：      
      其中 0 ~ 8 编辑九宫格行为 0 无行为 1 为放置方块 2 为推掉方块
      9存储此次行为消耗的R1方块数量
      10存储此次行为消耗的R2方块数量
      11存储此次行为消耗的武器数量
      12存储此次行为为R1执行还是R2执行
      13存储是红方还是蓝方
  * 状态编辑：
      0~8 九宫格状态，0为空，1为我方方块，2为敌方方块
---
* 设计的代价函数：
    得分 = 得分权重 * 方块得分 + 位置权重 * 位置得分 - 时间权重 * 所消耗时间 + 失分权重 * 对方失分
  * 方块得分：根据编辑的行为，按照真实值计算该行为对应的方块分数总和，第一层R1方块一个40，第二层R2方块一个70，第三层R2方块1个100
  * 位置得分：下九宫格正中心100，对角位置70，边边40
  * 消耗时间：通过机器人当前位置连续运动到他决策的所有点位时间之和，最终消耗时间按照消耗时间最长的机器人决定的，时间计算是按照梯形速度规划来预估的
  * 失分计算：根据戳掉对面方块的种类决定，第一层R1方块一个40，第二层R2方块一个70，第三层R2方块1个100
---
* 行为选择逻辑：
一： 提前存储好所有行为共204种搭配
  * 全部行为（目前当作放R1和武器捅对面独立）：
    1.  放1个 R1：0~2   共3种行为                                          
    2. 放1个 R1，1个 R2  R1 0~2   R2 3~8            共 3*6 = 18种行为     
    3. 放1个 R2 3~8                                 共6种行为
    4.  放2个 R2       34 35 36 37 38  45 46 47 48 56 57 58 67 68 78  共15种行为
    5.  放1个 R2 用 1个武器  共 6*8 = 48 种行为
    6.  放2个 R2 用 1个武器  共15*7 = 105种行为
    7.  用1个武器  0~8 共9种行为
  
二：首先提前过滤掉一些明显不合理的行为组合
三：根据代价函数比较得分，得分最高的为该次行为

---
* 权重选择逻辑:
* 权重参数按照四位，以0.01步进参数
* 通过生成三区的一定范围的随机位置，一定范围的R1,R2速度加速度参数，对对手设置几个固定参数，进行模拟训练，每种权重参数模拟100次，选出胜率最高的权重参数
---
实际效果：行为选择决策没有问题，但权重选择效果很差

#### 思路2：
* 文件：
      上位机：nine_square_strategy
      下位机：RC2026_H7_nine_square_strategy/RC26_LIB/RC_best_strategy.cpp
---

* 实现：
      R2未合体状态下：
          1.优先判断3，4，5三个格子哪个是空的且满足我方下这或敌方下这能大胜，然后根据机器人到对应点位的距离从小到打排序所将放置点位
          2.在1判断完的条件下判断中心格是否为空，若空优先放中心格
          3.判断剩下的位置是否有空，有空按照距离排序
      R2合体状态下：
          1.优先判断3,4,5,6,7,8六个格子哪个是空的且满足我方下这或敌方下这能大胜，然后根据机器人到对应点位的距离从小到打排序所将放置点位
          2.在1的基础上判断6，7，8是否为空，按照距离排序
          3.剩下的空格子按照距离排序

---
* 上位机按照此思路写了服务通讯的节点，客户端发送状态给服务端，服务端返回最优决策
* 下位机也是这个思路，接收九宫格状态，机器人位置状态，目标放置点位状态，判断选出最优决策，区别是这里不止最优决策，还按照优先级将其他决策排序列在一个数组里，方便最优决策失效或无法执行时，选择其他决策
